Fix Missing Booking URLs for SeatsAero Flight Deals

CRITICAL ISSUE:
The SeatsAero integration is working and returning flight deals, but all deals show "Booking unavailable" because booking URLs are not being generated.

PROBLEM ANALYSIS:
1. SeatsAero API response does not include booking URLs in the availability endpoint
2. The `extractBookingUrl()` method returns undefined because no booking URL fields exist in the response
3. Frontend displays "Booking unavailable" button instead of "Book Now" button
4. Users cannot complete bookings directly to airline websites

REQUIRED SOLUTION:
Generate booking URLs dynamically based on airline and flight details from the SeatsAero response.

IMPLEMENTATION NEEDED:

1. **Update `extractBookingUrl()` method in SeatsAeroPartnerService:**
Instead of looking for existing booking URL fields, construct URLs based on:
- Airline/carrier (from Route.Source)
- Origin and destination airports
- Flight date
- Cabin class
- Miles/points required

2. **Create airline-specific booking URL generators:**
